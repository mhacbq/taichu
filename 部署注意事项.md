# 太初命理网站 - 部署注意事项

## 一、环境要求

### 1. PHP 环境
- PHP 版本：8.0 或更高
- 必需扩展：pdo_mysql, json, mbstring, openssl, curl, fileinfo, redis（可选）

### 2. 数据库
- MySQL 版本：8.0 或更高（推荐 8.0+）
- 字符集：utf8mb4

### 3. Web 服务器
- Nginx 或 Apache
- 支持 URL 重写（伪静态）

---

## 二、文件结构

```
taichu-unified/
├── admin/              # 管理后台前端（Vue3）
├── backend/            # 后端 API（ThinkPHP 8）
│   ├── app/            # 应用代码
│   ├── config/         # 配置文件
│   ├── database.sql    # 数据库文件（重要！）
│   ├── public/         # 入口目录
│   ├── runtime/        # 缓存目录（需可写）
│   └── vendor/         # Composer 依赖
├── frontend/           # 用户端前端（Vue3）
├── database/           # 其他数据库文件
└── deploy/             # 部署相关
```

## 三、数据库部署步骤

### 方式一：使用 Navicat（推荐）

1. **创建数据库**
   - 打开 Navicat
   - 新建数据库，名称如 `taichu`
   - 字符集选择 `utf8mb4`，排序规则 `utf8mb4_unicode_ci`

2. **导入 SQL 文件**
   - 右键点击数据库 → 运行 SQL 文件
   - 选择 `backend/database.sql`
   - 点击开始，等待导入完成

### 方式二：使用 MySQL 命令行

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE taichu CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 导入数据
mysql -u root -p taichu < backend/database.sql
```

## 四、后端部署

### 1. 安装依赖（首次部署）

```bash
cd backend
composer install --no-dev
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env`，修改以下配置：

```ini
# 数据库配置（根据实际修改）
DB_HOST = localhost
DB_PORT = 3306
DB_NAME = taichu
DB_USER = root
DB_PASSWORD = your_password
DB_PREFIX = tc_

# 网站配置
APP_DEBUG = false
APP_URL = https://your-domain.com

# JWT密钥（随机生成，用于Token加密）
JWT_SECRET = your_random_secret_key_here

# 微信小程序配置（如需）
WECHAT_APPID = your_appid
WECHAT_SECRET = your_secret

# 微信支付配置（如需）
WECHAT_MCH_ID = your_mch_id
WECHAT_KEY = your_api_key

# 短信配置（如需）
SMS_PROVIDER = tencent
SMS_SECRET_ID = your_secret_id
SMS_SECRET_KEY = your_secret_key
SMS_SIGN = your_sign_name
```

### 3. 设置目录权限

Windows 下确保以下目录有写入权限：
- `backend/runtime/` - 缓存目录
- `backend/public/uploads/` - 上传目录（如不存在需创建）

### 4. 配置伪静态（Nginx）

```nginx
location / {
    if (!-e $request_filename) {
        rewrite ^(.*)$ /index.php?s=$1 last;
    }
}

location ~ \.php$ {
    fastcgi_pass 127.0.0.1:9000;
    fastcgi_index index.php;
    fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    include fastcgi_params;
}
```

### 5. 配置伪静态（Apache）

确保 `public/.htaccess` 文件存在：

```apache
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule ^(.*)$ index.php/$1 [QSA,PT,L]
</IfModule>
```

## 六、本地开发环境（phpStudy）

### 1. 启动服务
- 打开 phpStudy
- 启动 Apache/Nginx 和 MySQL

### 2. 创建网站

## 七、前端部署

### 1. 安装依赖

```bash
# 用户端
cd frontend
npm install

# 管理后台
cd admin
npm install
```

### 2. 配置 API 地址

编辑 `frontend/.env.production`：
```
VITE_API_BASE_URL=https://your-domain.com/api
```

编辑 `admin/.env.production`：
```
VITE_API_BASE_URL=https://your-domain.com/api
```

### 3. 构建部署

```bash
# 构建用户端
cd frontend
npm run build

# 构建管理后台
cd admin
npm run build
```

将 `dist` 文件夹内容部署到对应站点目录。

## 八、常见问题

### 1. 数据库连接失败
- 检查 `.env` 中的数据库配置
- 确认 MySQL 服务已启动
- 检查数据库用户名密码是否正确

### 2. 500 错误
- 检查 `runtime/` 目录是否有写入权限
- 查看 `runtime/log/` 下的错误日志
- 确认 PHP 版本 >= 8.0

### 3. 伪静态不生效
- 确认 Nginx/Apache 已开启 rewrite 模块
- 检查配置文件路径是否正确

### 4. 跨域问题
- 确认 API 地址配置正确
- 检查后端 `config/cors.php` 配置

## 十一、定时任务配置（重要）

系统需要配置定时任务来清理过期数据和生成统计数据。

### 可用命令

```bash
# 进入后端目录
cd backend

# 运行每日定时任务
php think daily:task

# 清理缓存
php think cache:clear [all|file|redis|log|temp]

# 安全检查
php think security:check
```

### Windows 计划任务配置

#### 方法一：使用 Windows 任务计划程序（推荐）

1. **打开任务计划程序**
   - 搜索"任务计划程序"并打开
   - 或运行 `taskschd.msc`

2. **创建基本任务**
   - 点击"创建基本任务"
   - 名称：`太初命理每日任务`
   - 描述：`清理过期数据、处理超时订单`

3. **设置触发器**
   - 选择"每天"
   - 开始时间：凌晨 3:00（建议）
   - 每隔：1 天

4. **设置操作**
   - 选择"启动程序"
   - 程序/脚本：`C:\php\php.exe`（你的PHP路径）
   - 添加参数：`think daily:task`
   - 起始于：`C:\www\taichu\backend`（你的项目路径）

5. **完成设置**
   - 勾选"打开属性对话框"
   - 点击完成

6. **高级设置**
   - 在属性窗口中选择"不管用户是否登录都要运行"
   - 勾选"使用最高权限运行"
   - 设置"如果任务运行时间超过"为 1 小时
   - 设置"如果任务失败，重新启动间隔"为 10 分钟

#### 方法二：宝塔面板定时任务配置（推荐宝塔用户）

**步骤一：进入计划任务**
1. 登录宝塔面板（`http://服务器IP:8888`）
2. 点击左侧菜单 **"计划任务"**

**步骤二：添加每日任务（清理过期数据）**

点击 **"添加计划任务"**，填写：

| 设置项 | 填写内容 |
|--------|----------|
| **任务类型** | `Shell脚本` |
| **任务名称** | `太初每日清理任务` |
| **执行周期** | `每天` |
| **执行时间** | `3时00分`（建议凌晨3点）|
| **脚本内容** | 见下方代码 |

**脚本内容：**
```bash
#!/bin/bash
cd /www/wwwroot/taichu/backend
php think daily:task
echo "[$(date '+%Y-%m-%d %H:%M:%S')] 每日任务执行完成" >> /www/wwwroot/taichu/backend/runtime/cron.log
```

**步骤三：添加每周任务（清理日志缓存）**

再添加一个任务：

| 设置项 | 填写内容 |
|--------|----------|
| **任务类型** | `Shell脚本` |
| **任务名称** | `太初清理日志缓存` |
| **执行周期** | `每周` → 选择 `周日` |
| **执行时间** | `4时00分` |
| **脚本内容** | 见下方代码 |

**脚本内容：**
```bash
#!/bin/bash
cd /www/wwwroot/taichu/backend
php think cache:clear log
echo "[$(date '+%Y-%m-%d %H:%M:%S')] 日志清理完成" >> /www/wwwroot/taichu/backend/runtime/cron.log
```

**步骤四：添加每月任务（安全检查）**

| 设置项 | 填写内容 |
|--------|----------|
| **任务类型** | `Shell脚本` |
| **任务名称** | `太初安全检查` |
| **执行周期** | `每月` → 选择 `1日` |
| **执行时间** | `5时00分` |
| **脚本内容** | 见下方代码 |

**脚本内容：**
```bash
#!/bin/bash
cd /www/wwwroot/taichu/backend
php think security:check >> /www/wwwroot/taichu/backend/runtime/security_check.log 2>&1
echo "[$(date '+%Y-%m-%d %H:%M:%S')] 安全检查完成" >> /www/wwwroot/taichu/backend/runtime/cron.log
```

**步骤五：测试任务**

添加完成后：
1. 在任务列表中找到刚添加的任务
2. 点击右侧的 **"执行"** 按钮立即测试
3. 点击 **"日志"** 按钮查看执行结果

**步骤六：查看执行日志**

执行完成后，可在以下位置查看日志：
- 每日任务日志：`/www/wwwroot/taichu/backend/runtime/cron.log`
- 安全检查日志：`/www/wwwroot/taichu/backend/runtime/security_check.log`

**宝塔定时任务汇总表**

| 任务名称 | 周期 | 时间 | 作用 |
|---------|------|------|------|
| 太初每日清理任务 | 每天 | 03:00 | 清理过期验证码、取消超时订单、生成明日运势 |
| 太初清理日志缓存 | 每周日 | 04:00 | 清理7天前的日志文件 |
| 太初安全检查 | 每月1号 | 05:00 | 检查系统安全配置 |

**常见问题**

1. **如果执行失败，可能是 PHP 路径问题**
   
   修改脚本，使用宝塔 PHP 完整路径：
   ```bash
   # 查看 PHP 路径（宝塔终端执行）
   which php
   
   # 例如返回 /www/server/php/80/bin/php
   # 则脚本改为：
   /www/server/php/80/bin/php think daily:task
   ```

2. **查看 PHP 版本路径**
   - 宝塔面板 → 左侧"软件商店" → 已安装
   - 找到 PHP-8.0 或其他版本
   - 点击"设置" → 查看安装路径

### 定时任务说明

| 任务 | 频率 | 说明 |
|------|------|------|
| `daily:task` | 每天 | 清理过期验证码、取消超时订单、清理日志、生成运势、生成统计 |
| `cache:clear log` | 每周 | 清理7天前的日志文件 |
| `security:check` | 每月 | 运行安全检查 |

### 手动测试定时任务

在部署后，建议先手动运行测试：

```bash
cd backend

# 测试每日任务
php think daily:task

# 测试缓存清理
php think cache:clear all

# 测试安全检查
php think security:check
```
