## 🔴 严重问题

### 1. 短信验证码发送失败无降级方案

**位置**: `backend/app/service/SmsService.php`

**问题描述**:
当腾讯云短信服务配置不正确或API调用失败时，用户将无法收到验证码，导致无法注册/登录。

**当前代码**:
```php
// 获取短信配置
$config = SmsConfig::getTencentConfig();
if (!$config) {
    return ['success' => false, 'message' => '短信服务未配置'];
}
```

**影响**: 用户无法完成注册和登录流程

**建议修复**:
- 添加模拟发送模式（开发环境）
- 添加备用短信服务商
- 添加邮件验证码备选方案

---

### 2. 积分扣除后事务未完整处理

**位置**: `backend/app/model/User.php`

**问题描述**:
在 `deductPoints` 方法中，如果积分扣除成功但后续操作（如记录积分历史）失败，会导致数据不一致。

**当前代码**:
```php
public function deductPoints(int $points): bool
{
    // ... 事务处理积分扣除 ...
    
    // 此方法只处理积分扣除，不处理后续记录
    // 调用方需要自行处理记录逻辑
}
```

**影响**: 积分变动记录可能丢失

**建议修复**:
- 将积分变动记录也纳入同一事务
- 或使用事件监听机制确保记录完整性

---

### 3. AI分析接口缺少输入验证

**位置**: `backend/app/controller/AiAnalysis.php`

**问题描述**:
AI分析接口接收 `bazi` 参数，但没有验证数据格式，可能导致AI API调用失败或返回错误结果。

**当前代码**:
```php
public function analyzeBazi(Request $request)
{
    $baziData = $request->param('bazi');
    
    if (empty($baziData)) {
        return json(['code' => 400, 'message' => '八字数据不能为空']);
    }
    // 缺少对 $baziData 结构的验证
}
```

**影响**: 可能导致AI服务错误、不必要的API调用费用

**建议修复**:
```php
// 验证八字数据结构
$requiredKeys = ['year', 'month', 'day', 'hour'];
foreach ($requiredKeys as $key) {
    if (!isset($baziData[$key]) || !isset($baziData[$key]['gan']) || !isset($baziData[$key]['zhi'])) {
        return json(['code' => 400, 'message' => '八字数据格式不正确']);
    }
}
```

## 🟠 中等问题

### 4. 塔罗历史记录仅保存在本地

**位置**: `frontend/src/views/Tarot.vue`, `frontend/src/views/Profile.vue`

**问题描述**:
塔罗占卜记录只保存在 `localStorage`，用户换设备或清除浏览器数据后会丢失。

**当前代码**:
```javascript
// Tarot.vue
localStorage.setItem('tarot_last_result', JSON.stringify({...}))

// Profile.vue
const saved = JSON.parse(localStorage.getItem('tarot_saved') || '[]')
```

**影响**: 用户体验不完整，数据无法跨设备同步

**建议修复**:
- 创建塔罗历史记录表
- 后端保存用户塔罗记录
- 与八字历史统一展示

---

### 5. 支付成功回调未处理订单状态

**位置**: `backend/app/controller/Payment.php`, `backend/app/controller/Alipay.php`

**问题描述**:
支付回调方法 `notify` 存在但缺少完整的订单状态处理和幂等性验证。

**当前代码**:
```php
public function notify()
{

    // 需要处理订单状态更新
    // 需要添加幂等性检查（防止重复处理）
```

**建议修复**:

### 6. 缺少数据备份机制

**位置**: 全局

**问题描述**:

## 检查总结

本次检查发现的问题主要集中在：

1. **容错性不足**: 短信服务、支付回调缺少降级和幂等性处理
2. **数据一致性**: 积分记录、塔罗历史数据存储不完整
3. **性能优化**: 缺少缓存、CDN等优化手段
4. **可观测性**: 日志、监控不够完善

建议按优先级逐步修复，确保系统稳定性和用户体验。
