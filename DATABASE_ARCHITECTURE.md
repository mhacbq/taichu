## 📐 数据库总体架构

```

│
├─ 🔐 用户与认证系统 (9张表)

│  ├─ user_balances                  ← 余额统计

│
├─ 💰 积分与财务系统 (8张表)

│

│  ├─ tc_payment_config              ← 支付配置

│
├─ 🎴 命理功能表 (12张表)
│  ├─ 八字系统
│  │  ├─ bazi_records                ← 八字排盘记录

│  │  └─ tc_tarot_record             ← 占卜记录
│  ├─ 六爻系统
│  │  └─ tc_liuyao_record            ← 六爻占卜记录

│     ├─ tc_article_category         ← 知识库分类
│     └─ tc_article                  ← 知识库文章
│
├─ 📝 内容管理系统 (7张表)
│  ├─ pages                          ← 页面主表
│  ├─ page_versions                  ← 页面版本
│  ├─ page_drafts                    ← 页面草稿
│  ├─ page_recycle                   ← 页面回收站
│  ├─ faqs                           ← 常见问题
│  ├─ site_contents                  ← 网站内容
│  └─ testimonials                   ← 用户见证
│
├─ 🤖 AI与配置 (3张表)
│  ├─ ai_prompts                     ← AI提示词
│  ├─ question_templates             ← 问题模板
│  └─ daily_fortune_templates        ← 每日运势模板
│
├─ 📊 运营与日志 (4张表)
│  ├─ operation_logs                 ← 操作日志
│  ├─ feedback                       ← 用户反馈
│  ├─ tc_share_log                   ← 分享记录
│  └─ tc_checkin_log                 ← 签到日志
│
├─ 📁 内容存储 (1张表)
│  └─ upload_files                   ← 文件记录
│
├─ 🔍 SEO与搜索 (6张表，可选)
│  ├─ tc_seo_config                  ← SEO页面配置
│  ├─ tc_seo_keywords                ← SEO关键词
│  ├─ tc_seo_indexed_pages           ← 页面收录
│  ├─ tc_seo_submissions             ← 提交记录
│  ├─ tc_seo_traffic_daily           ← 流量统计
│  └─ tc_seo_robots                  ← robots配置
│
└─ 🎯 系统配置 (1张表)
   └─ system_config                  ← 系统配置
```

## 🔄 数据流向图

### 1. 用户注册与积分初始化流程

```
┌────────────────┐
│  用户注册      │
│  (手机号)      │
└────────┬───────┘
         │
         ▼
┌────────────────────────────┐
│ 1. 验证短信验证码         │
│    查表: tc_sms_code       │
└────────┬───────────────────┘
         │
         ▼
┌────────────────────────────┐
│ 2. 创建用户账户           │
│    写入: users             │
│    - user_id (自动生成)    │
│    - phone                 │
│    - nickname              │
│    - avatar                │
│    - points (初始值)       │
│    - created_at            │
└────────┬───────────────────┘
         │
         ▼
┌────────────────────────────┐
│ 3. 初始化用户余额         │
│    写入: user_balances     │
│    - user_id              │
│    - total_points: 初始积分│
└────────┬───────────────────┘
         │
         ▼
┌────────────────────────────┐
│ 4. 记录注册任务           │
│    写入: tc_task_log       │
│    - task_type: register   │
│    - points: 奖励积分      │
└────────┬───────────────────┘
         │
         ▼
┌────────────────────────────┐
│ ✅ 注册完成               │
│ 用户获得初始积分          │
└────────────────────────────┘
```

### 2. 八字排盘与AI解读流程

```
┌────────────────┐
│  用户排盘八字   │
│  输入: 出生信息  │
└────────┬───────┘
         │
         ▼
┌────────────────────────────┐
│ 1. 检查积分余额           │
│    查表: user_balances     │
│    需要: points_cost_bazi  │
└────────┬───────────────────┘
         │
         ▼
┌─────────────────────────────────┐
│ 2. 调用八字排盘算法             │
│    查表: system_config         │
│    获取配置参数                 │
└────────┬────────────────────────┘
         │
         ▼
┌────────────────────────────┐
│ 3. 生成八字排盘结果       │
│    调用: AI/算法引擎       │
└────────┬───────────────────┘
         │
         ▼
┌────────────────────────────────┐
│ 4. 调用AI进行深度分析         │
│    查表: ai_prompts           │
│    使用: bazi分析提示词        │
│    → ChatGPT/Claude API        │
└────────┬────────────────────────┘
         │
         ▼
┌────────────────────────────┐
│ 5. 保存排盘结果           │
│    写入: bazi_records      │
│    - user_id              │
│    - name                 │
│    - birth_date           │
│    - bazi (JSON)          │
│    - analysis             │
│    - ai_analysis          │
│    - points_cost: 30      │
└────────┬───────────────────┘
         │
         ▼
┌────────────────────────────┐
│ 6. 扣费积分               │
│    写入: points_records    │
│    - user_id              │
│    - type: reduce         │
│    - points: -30          │
│    - action: bazi_chart   │
│    - created_at           │
└────────┬───────────────────┘
         │
         ▼
┌────────────────────────────┐
│ 7. 更新用户余额           │
│    更新: user_balances     │
│    - points -= 30         │
└────────┬───────────────────┘
         │
         ▼
┌────────────────────────────┐
│ ✅ 排盘完成               │
│ 用户可查看结果和分享      │
└────────────────────────────┘
```

### 3. VIP充值与订单流程

```
┌─────────────────────────┐
│  用户点击充值            │
│  选择: VIP套餐(68元/月)   │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────────────┐
│ 1. 创建充值订单                 │
│    写入: recharge_orders        │
│    - order_no: 生成唯一订单号   │
│    - user_id                   │
│    - amount: 68.00             │
│    - status: pending           │
│    - created_at                │
└────────┬────────────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ 2. 读取支付配置                  │
│    查表: tc_payment_config       │
│    - type: wechat_jsapi         │
│    - app_id                    │
│    - mch_id                    │
│    - api_key                   │
└────────┬─────────────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ 3. 调用微信支付API               │
│    上传订单信息                  │
│    微信返回: prepay_id           │
└────────┬─────────────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ 4. 前端调起支付               │
│    用户完成支付                  │
└────────┬─────────────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ 5. 微信回调通知                  │
│    后端接收支付成功通知          │
└────────┬─────────────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ 6. 更新充值订单                  │
│    更新: recharge_orders         │
│    - status: paid              │
│    - pay_order_no: 微信订单号   │
│    - paid_at: 支付时间          │
└────────┬─────────────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ 7. 创建VIP订单                   │
│    写入: tc_vip_order            │
│    - order_no                  │
│    - user_id                   │
│    - vip_level: 1              │
│    - amount: 68.00             │
│    - status: paid              │
│    - vip_start_at: 现在         │

         │
         ▼
┌──────────────────────────────────┐
│ 8. 添加积分奖励                  │
│    写入: points_records          │
│    - type: add               │
│    - action: vip_purchase       │

└────────┬─────────────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ 9. 更新用户VIP等级               │
│    更新: users                   │
│    - vip_level: 1              │
│    - updated_at                │
└────────┬─────────────────────────┘
         │
         ▼
┌──────────────────────────────────┐
│ ✅ 支付完成                      │
│ 用户升级为VIP，享受功能解锁     │
└──────────────────────────────────┘
```

## 🗂️ 表存储大小估算

```
假设：系统运行2年，10万注册用户

┌──────────────────────────────────┬──────────┬─────────┐
│ 表名                             │ 预估行数 │ 大小    │
├──────────────────────────────────┼──────────┼─────────┤
│ users                            │ 100K     │ ~10MB   │
│ points_records                   │ 2M       │ ~150MB  │ ⚠️ 需要分库分表
│ recharge_orders                  │ 50K      │ ~5MB    │
│ bazi_records                     │ 300K     │ ~50MB   │
│ operation_logs                   │ 1M       │ ~100MB  │ ⚠️ 需定期清理
│ tc_tarot_record                  │ 500K     │ ~80MB   │
│ tc_checkin_log                   │ 1M       │ ~80MB   │
│ pages                            │ 50       │ <1MB    │
│ tc_article                       │ 1K       │ ~5MB    │
│ tc_almanac                       │ 10K      │ ~2MB    │
│ (其他小表)                       │ 合计     │ ~30MB   │
├──────────────────────────────────┼──────────┼─────────┤
│ 总计                             │ ~5.5M行  │ ~500MB  │
└──────────────────────────────────┴──────────┴─────────┘

💡 优化建议：
   1. 定期清理/归档 operation_logs（可保留1年）
   2. 对 points_records 考虑分库分表（按用户ID）
   3. 使用缓存减少数据库查询（Redis）
   4. 建立合适的索引以加快查询
```
