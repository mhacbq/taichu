# 本地启用

本项目已提供一键本地启动脚本：`start-local.ps1`（位于网站根目录）。

## 1. 前置条件

- 已安装 Docker Desktop（并可执行 `docker compose`）
- Windows PowerShell 可用
- （可选）若要启动前端/后台管理端，需安装 Node.js + npm

## 2. 一键启动（推荐）

在项目根目录执行：

```powershell
Set-Location "C:\Users\v_boqchen\WorkBuddy\Claw\taichu-unified"
.\start-local.ps1
```

默认行为：

- 执行 `backend/docker-compose.yml` 的 `docker compose up -d --build`，确保后端容器吃到最新代码
- 启动 `backend/docker-compose.yml` 中的 `mysql + backend`
- 自动进行健康检查：`http://localhost:8080/api/health`
- 后端容器启动时会自动补跑 `database/20260317_create_admin_users_table.sql`、`database/20260317_create_shensha_table.sql`、`database/20260317_create_knowledge_tables.sql` 等幂等补丁，旧数据卷也能补齐后台缺表

## 3. 常用参数

### 3.1 首次初始化或需要重置数据库

```powershell
.\start-local.ps1 -ResetData
```

说明：会执行 `docker compose down -v`，清空 MySQL 数据卷后再重建。

### 3.2 不重新构建镜像（加快启动）

```powershell
.\start-local.ps1 -NoBuild
```

### 3.3 同时启动前端与管理端

```powershell
.\start-local.ps1 -WithFrontend -WithAdmin
```

说明：会在新的 PowerShell 窗口中执行：

- `frontend`: `npm install; npm run dev`
- `admin`: `VITE_PROXY_TARGET=http://localhost:8080; npm install; npm run dev`

## 4. 访问地址

- 后端 API：`http://localhost:8080`
- 健康检查：`http://localhost:8080/api/health`
- 独立后台（启用 `-WithAdmin` 时）：`http://localhost:3001`，其 `/api/*` 默认代理到 `http://localhost:8080`

## 5. 停止服务

```powershell
Set-Location "C:\Users\v_boqchen\WorkBuddy\Claw\taichu-unified"
.\stop-local.ps1
```

清理数据卷并停止（会清空本地 MySQL 数据）：

```powershell
.\stop-local.ps1 -RemoveVolumes
```

## 6. 一键重启

```powershell
Set-Location "C:\Users\v_boqchen\WorkBuddy\Claw\taichu-unified"
.\restart-local.ps1
```

常用组合：

```powershell
.\restart-local.ps1 -NoBuild
.\restart-local.ps1 -ResetData
.\restart-local.ps1 -WithFrontend -WithAdmin
```

## 7. 最小验证步骤

```powershell
curl.exe -i http://localhost:8080/api/health
curl.exe -i http://localhost:8080/api/stats/home
```

- `api/health` 返回 `200` 表示后端已启动
- `api/stats/home` 返回业务 JSON 表示数据库连通且初始化完成

## 8. 常见问题

### 8.1 `SQLSTATE[HY000] [2002] Connection refused`

通常是 MySQL 未就绪。可先重试：

```powershell
.\start-local.ps1 -ResetData
```

### 8.2 缺表报错（如 `tc_admin` / `tc_shensha` / `tc_article` 不存在）

先执行一次：

```powershell
.\restart-local.ps1
```

当前后端容器会在启动时自动补跑后台补丁 SQL；若基础表本身损坏或首次初始化不完整，再执行 `-ResetData` 触发完整重建。

### 8.3 8080 端口被占用

先停止占用端口的容器或进程，再重启脚本。
